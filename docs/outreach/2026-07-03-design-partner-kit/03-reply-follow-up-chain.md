# 03 — Follow-up chain and the warm-yes reply

**Cadence for this cohort:** first touch → **+5 days** → **+12 days** → **+21 days** → close the row. Three follow-ups, then stop. (The T+3/7/14/21 sequence in the packet library is the general-purpose ladder; this founder-sent cohort runs the slower 5/12/21 spacing deliberately — five prospects, high care, no pressure plays.)

Each follow-up replies in the same thread (subject stays `re:` the original). Each one adds something new — a follow-up that just says "bumping this" spends trust and buys nothing. All the first-touch rules apply: token scan before send, no invented details, soft ask only.

---

## Follow-up 1 — day 5 (short nudge, one new concrete thing)

```
Hi {{first-name}},

Following up on my note from last week. One thing I didn't include:
I've recorded a 6-minute walkthrough of the product working on a
demonstration workspace — real product, synthetic data, labeled as
such on screen. If a call is a bigger step than you want right now,
I'll send the recording instead. Just say the word.

Either way — worth 20 minutes this week or next?

Conner
```

*~65 words. The recorded walkthrough is the proof-ladder substitute from the design-partner program doc — build it once, offer it everywhere. If the recording doesn't exist yet, this follow-up waits until it does; do not offer what isn't built.*

## The warm-yes reply — book the call without the placeholder failure

**Trigger:** any positive reply ("sure," "tell me more," "how does Thursday look?"). Send within 4 business hours if possible — reply speed is the first proof of the service ethos.

**The old failure mode:** every prior outreach draft dead-ended at a `{{CALENDLY_LINK}}` placeholder that was never wired. The rule now: **the booking link is optional, the offered times are not.** Two concrete windows go in every warm-yes reply. If `{{booking-url}}` isn't live yet (the send-path PR is wiring the real URL), delete that line and book by reply — the email works without it.

```
Hi {{first-name}},

Good to hear from you. Two windows that work on my end this week:

- {{day-1}}, {{time-range-1}} ET
- {{day-2}}, {{time-range-2}} ET

If either works, reply with one and I'll send a calendar invite with
a call link. If it's easier to pick from my calendar directly:
{{booking-url}}

So you know what you're saying yes to: 20 minutes, most of it me
asking how your week actually runs. I'll show you one thing — a
draft sitting in an approval queue — and we'll both decide if
there's a next step. No slides.

Conner
```

**Pre-send guard for this template specifically:**
- [ ] Both time windows are real and currently open on the calendar.
- [ ] `{{booking-url}}` either resolves to the live booking page or the line is deleted. **The token itself must never reach a prospect.**
- [ ] Calendar invite goes out the same day they pick a slot, with the call link in the invite body.
- [ ] CRM row → CALL-BOOKED with the date, and a briefing doc (document 04) is started from the template.
