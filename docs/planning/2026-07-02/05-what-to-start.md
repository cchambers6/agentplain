# Start list — the three highest-leverage moves we aren't making

Not a wish list. Each move is scoped, names why it hasn't happened yet, and states what it
unblocks. Together they consume roughly the capacity the stop list frees.

## Move 1 — Fire the activation fix wave (the five fixes every July document already names)

**What:** one fleet wave, sequential PRs, drawn from kaizen master §3 and the audit top-20:

1. **Connect-button dead ends** — wire the FUB/Sierra api-key credential form to the tile CTA;
   flip TaxDome/Karbon to honest `coming-soon` tiles (the S-effort truth fix; the L-effort live
   adapters wait). This repairs the advertised 5-minute first-value path for the beachhead
   vertical — the single most valuable S-effort change in the company.
2. **Saved-time writers + guarantee leak** — add writers to the 4 uncovered calibrated actions
   and route walk-away refunds through human review (audit 9 P0-1). Stops a live money leak and
   makes the case-study metric truthful before pilot week 1.
3. **`/how-it-works` redirect fix** (audit 1 P0) — the front door for every outreach link.
4. **Server-side CI floor** — `pr-checks.yml` running typecheck + invariant tests + brand/
   voice/vendor-name/claims gates. Ends the gates-local-only / HUSKY=0 pattern (kaizen F2) and
   locks the vendor-invisible rule permanently.
5. **Wire the meters** — `stampSessionCost` call sites + the librarian-rollup executor, so
   budget-state stops reading NULL.

**Why it hasn't happened:** every session that could have fired it got assigned another
analysis layer instead. The work is named, S/M effort, fully fleet-executable, zero Conner
dependencies.

**What it unblocks:** the "market-ready" half of the prod-key un-pause gate; honest pilot
measurement; the CI substrate every later rule depends on. This is the wave that converts
three weeks of audits into product.

## Move 2 — Ship the Conner Decision Pack (make the human bottleneck one sitting wide)

**What:** a single PR that populates `memory/data/conner-queue.yaml` (empty since creation)
with every pending Conner-only decision, each carrying a recommended default, time estimate,
SLA class, and what it blocks:

- **Entity + counsel confirmation** (blocker-class; blocks signing anything — the biggest
  silent gap, see 02.1)
- The **6 kaizen contradictions** (prod-key finance question, cohort 3–5 vs 10, "built on
  Claude" vs vendor-invisible, /security absolutes, portal fund-or-flag, Librarian cadence)
- The **6 sales-plan blocking decisions** (cohort confirm, founder-time calendar commitment,
  ~$100–160/mo GTM accounts, key un-pause target date, founder bio/on-record identity,
  pre-outreach copy fixes)

Plus the morning-brief wiring so the queue actually reaches him.

**Why it hasn't happened:** classic seam failure — every session appended decisions to its own
document's §5 and assumed someone else aggregates. Nobody owns the aggregation; the Librarian's
queue file exists but nothing writes to it (the "meter with no wire" pattern, applied to the
founder himself).

**What it unblocks:** literally everything human-gated. The plan's honest critical path runs
through ~5 Conner actions totaling maybe two focused days; today they are scattered across
four documents. This move is the cheapest schedule compression available to the business.

## Move 3 — Define "profitable," in YAML, with wired inputs

**What:** a one-page ratified definition plus the machine version in `memory/data/`:

- A **milestone ladder**, not a single boolean — e.g. M1: first signed design partner; M2:
  first activated pilot; M3: first paid conversion; M4: MRR ≥ 4-week trailing fleet spend +
  infra (true "profitable"). Exact thresholds are Conner's call (it's in the Decision Pack);
  the fleet's job is to propose the ladder and wire the inputs.
- Inputs that actually flow: session cost stamping (move 1.5), Stripe MRR read, infra
  line items.

**Why it hasn't happened:** "profitable" felt self-evident, so every directive used the word
and no artifact defined it. The 9-track loop directive finally made the omission operational:
a machine is being told to run until a flag flips that nothing can compute.

**What it unblocks:** a legitimate stop/scale condition for every loop; a data-driven Jul 6
re-tier decision (the post-Jul-7 economics cliff, see 02.6); and the finance-vs-policy
prod-key contradiction gets adjudicated against a number instead of vibes.

---

**Sequencing between the three:** they don't conflict — Move 1 is fleet-wide, Move 2 is one
session today, Move 3 is one session this week. If forced to pick one: **Move 2.** Move 1
makes the product ready; only Move 2 makes the *founder* ready, and the founder is the binding
constraint (direction verdict, precondition 2).
