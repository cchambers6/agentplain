# Cross-department coordination + handoff matrix

Turn N independent plans into one plan-of-record: a single sequence, a dependency matrix with named producers/consumers, and the ONE bottleneck the whole window hinges on.

## Procedure

1. **Unify** into a 14-day plan-of-record: Days 0–2 (merge decision docs, park conflicts, close stale PRs) → Days 2–7 (a *sequential* fix wave + a parallel human-decision pack) → Days 7–14 (verify, measure, hold the line).
2. **Handoff matrix** — for each function, "what I need from others" ↔ "what others need from me." Every dependency names both ends.
3. **Conflicts** — list every place two plans disagree; resolve to one, or escalate the residual to the human (don't silently pick).
4. **Consolidated decision queue** — merge every plan's forced decision into ONE queue.
5. **Critical path** — name the single bottleneck the whole plan depends on.

## Rules

- **Sequential, not parallel, for overlapping work** — the fix wave is 5–6 sequential landings.
- **One bottleneck, named** — the headline is the single thing everything hinges on, not a list.
- **Surface residual conflicts** — a genuine disagreement (e.g. two docs specifying different daily caps) is escalated for a ruling, not reconciled by fiat.
- **Every dependency is owned** — an unowned handoff is a dropped ball.

## Origin

The coordination pass of 2026-07-03 (`docs/departments/2026-07-03/COORDINATION/`, PR #368): reconciled ten head plans into one plan-of-record whose entire two weeks reduced to a single bottleneck — un-pausing the production key.
